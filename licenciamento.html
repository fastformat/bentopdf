AGPL Requisitos

Ao usar o FastFormat sob a AGPL-3.0, você deve cumprir com os seguintes requisitos:

Linha do Produtor & Direitos Autorais

Você deve mencionar nosso código aberto e incluir detalhes da licença AGPL nos metadados do PDF. Você também deve reter a linha do produtor em todos os PDFs criados ou modificados usando nosso código aberto.

Modificações de Código

Qualquer modificação em nosso código aberto deve ser divulgada sob a AGPL. Isso inclui compartilhar o código modificado com quaisquer usuários interagindo com ele.

Apenas Ambientes Compatíveis com AGPL

O uso de nossa Edição da Comunidade de código aberto deve ser limitado a ambientes compatíveis com a AGPL, onde todos os requisitos da AGPL são totalmente respeitados.

Divulgação do Código-Fonte

Você não pode implantar nosso código aberto como parte de um aplicativo ou serviço baseado em servidor sem divulgar o código-fonte completo do seu próprio aplicativo sob a AGPL para quaisquer usuários interagindo com ele.

O que é Compatível com a AGPL?

Compatível com a AGPL

  • Projetos de código aberto onde você compartilha seu código-fonte completo publicamente
  • Uso pessoal onde você não está distribuindo ou hospedando como um serviço
  • Ferramentas hospedadas publicamente onde você fornece o código-fonte completo (incluindo modificações) sob a AGPL
  • Projetos educacionais ou de pesquisa que tornam todo o código disponível publicamente

NÃO Compatível com a AGPL

  • Aplicativos SaaS proprietários/de código fechado
  • Ferramentas internas da empresa onde o código-fonte não é compartilhado com os usuários
  • Produtos comerciais que não divulgam o código-fonte completo
  • Serviços de rede que não fornecem acesso ao código-fonte para os usuários
  • Qualquer aplicativo onde você queira manter seu código-fonte privado

⚠️ Esses casos de uso exigem uma licença comercial.

Licenciamento FAQs

Se você estiver usando o FastFormat em um site público e não modificou o código-fonte, não precisará de uma licença comercial. No entanto, se você modificar o FastFormat de qualquer forma e disponibilizá-lo para os usuários pela rede, a AGPLv3 exige que você forneça seu código-fonte modificado a esses usuários. Se você deseja manter suas modificações em código fechado ou se estiver integrando o FastFormat em um produto proprietário ou comercial, precisará de uma licença comercial. Entre em contato conosco para discutir seu caso de uso específico.

Se a sua ferramenta interna for de código fechado e não for compartilhada publicamente, você precisará de uma licença comercial. A licença AGPL-3.0 exige que você disponibilize o código-fonte de qualquer serviço acessível pela rede usando o FastFormat para seus usuários. Se você não pode ou não quer fazer isso, uma licença comercial é a escolha certa.

Se você modificar o FastFormat e implementá-lo como um serviço de rede (por exemplo, um aplicativo da web), deverá disponibilizar seu código-fonte modificado sob a AGPL-3.0. Se você não deseja compartilhar suas modificações, deve obter uma licença comercial.

Entrega & Modelo de Licenciamento

  • Nenhuma chave de licença é exigida ou fornecida. O FastFormat não usa intencionalmente chaves de licença para otimizar a implantação e reduzir o atrito operacional para os usuários.
  • Após a compra, os usuários recebem um arquivo ZIP contendo o código-fonte e binários completos do FastFormat.
  • O código-fonte fornecido é idêntico ao código publicado em nosso repositório de código aberto público.
  • Sem reembolsos. Sob nenhuma circunstância será feito um reembolso após a compra.

Repositório do GitHub: github.com/alam00000/bentopdf

Faturamento

  • Nós usamos o Polar para pagamentos, que envia faturas automaticamente via e-mail após a compra.
  • O Polar lida com faturas de IVA (VAT) para empresas em regiões aplicáveis.
  • For faturas de IVA (VAT), select "Estou comprando como empresa" durante o checkout e insira seu endereço de cobrança e número de Imposto/IVA.

Componentes AGPL - Não Integrados

FastFormat não integra bibliotecas de processamento licenciadas sob a AGPL. Os componentes a seguir devem ser configurados separadamente através de Configurações Avançadas se você deseja usar seus recursos:

  • PyMuPDF (AGPL-3.0)
  • Ghostscript (AGPL-3.0)
  • CoherentPDF / CPDF (AGPL-3.0)

Para habilitar os recursos alimentados por essas bibliotecas:

  • 1. Navegue para Configurações Avançadas no FastFormat
  • 2. Configure a URL para cada módulo WASM que você precisa
  • 3. Você pode hospedar seus próprios arquivos, usar um proxy WASM, ou usar qualquer CDN compatível

A licença comercial cobre apenas o próprio código do FastFormat. Ela não ignora o licenciamento AGPL desses componentes. Os usuários devem cumprir com os termos da AGPL v3 para esses componentes.
